## Review Checklist

Go through each item and note any issues:

### 1. Correctness
- Does the fix actually solve the reported bug?
- Are there edge cases the fix does not handle?
- Could the fix introduce a regression?

### 2. TypeScript
- Are all types explicit and correct?
- No use of `any` unless justified?
- No TypeScript errors would be introduced?

### 3. React Patterns
- Are hooks used correctly? (correct dependency arrays, no stale closures)
- No unnecessary re-renders introduced?
- State updates are immutable (no direct mutation of arrays/objects)?

### 4. Code Quality
- Is the fix minimal and focused? (No unrelated changes)
- Follows the existing naming conventions and code style?
- No leftover debug code (`console.log`, commented-out code)?

### 5. Tests
- If tests exist for the modified code, are they updated?
- Does the fix warrant a new test?

## Rules
- Be specific — vague feedback like "improve variable names" is not helpful
- Only request changes for real issues, not style preferences
- If the fix is correct but could be improved, mention it in SUMMARY but still APPROVE
- Do NOT suggest refactoring unrelated code
